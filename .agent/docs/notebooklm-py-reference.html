What Is It?

notebooklm-py is an unofficial Python library and CLI tool that reverse-engineers Google NotebookLM's internal RPC API. It gives you full programmatic access to every NotebookLM feature — and even exposes capabilities that the web UI doesn't offer.

Not affiliated with Google. This uses undocumented internal endpoints. Google can change or break these at any time without notice. Best used for prototypes, research tools, and personal automation — not production services at scale.

Authentication — The Big Question
You do NOT pay separately. It runs under your Google account. notebooklm-py authenticates as YOU — it uses real Google session cookies from a browser login, so all generation costs, rate limits, and quotas are tied to your personal Google (or Google Workspace) account. There is no API key, no credit system, and no billing beyond your existing Google plan.

How Authentication Actually Works

When you run notebooklm login, the library opens a real Chromium browser window. You sign in normally to your Google account. When complete, it saves your session cookies to a local file — ~/.notebooklm/storage_state.json. Every subsequent command re-uses those cookies to act as you.

  1. Run notebooklm login → a Chromium browser opens
  2. Log in to your Google account normally in that browser
  3. Press Enter in the terminal when done
  4. Session cookies are saved to disk at ~/.notebooklm/storage_state.json
  5. All future commands silently reuse these cookies
  6. If cookies expire, run notebooklm login again

Auth Storage Locations (Priority Order)

PrioritySourceBest For
1 (highest)Explicit path arg: from_storage("/path/file.json")Multi-account setups
2Environment variable: NOTEBOOKLM_AUTH_JSONCI/CD pipelines, GitHub Actions
3$NOTEBOOKLM_HOME/storage_state.jsonCustom home directory
4 (default)~/.notebooklm/storage_state.jsonNormal local development

Token Refresh

CSRF tokens auto-refresh transparently on 401/403 errors. However, if your session cookies expire (typically days to weeks), you must re-run notebooklm login. For CI/CD, export your storage_state.json content as a secret and set it in NOTEBOOKLM_AUTH_JSON.

Your Google account is the billing layer. Free Google accounts have NotebookLM limits (50 notebooks, 50 sources each, ~50 queries/day). Google AI Pro/Ultra subscribers get significantly higher quotas. The library has no way to bypass these limits.

Python API

The Python API is fully async using asyncio and the async with context manager pattern. Always wrap the client in async with to properly manage HTTP connections.

Core Pattern

import asyncio
from notebooklm import NotebookLMClient

async def main():
    # Auth from saved cookie file (run `notebooklm login` first)
    async with await NotebookLMClient.from_storage() as client:

        # Create notebook + add sources
        nb = await client.notebooks.create("My Research")
        await client.sources.add_url(nb.id, "https://example.com", wait=True)

        # Chat
        result = await client.chat.ask(nb.id, "Summarize this")
        print(result.answer)

        # Generate podcast, wait for it
        status = await client.artifacts.generate_audio(nb.id, instructions="make it engaging")
        await client.artifacts.wait_for_completion(nb.id, status.task_id)
        await client.artifacts.download_audio(nb.id, "podcast.mp3")

        # Generate mind map (synchronous — returns immediately)
        await client.artifacts.generate_mind_map(nb.id)
        await client.artifacts.download_mind_map(nb.id, "mindmap.json")

asyncio.run(main())

Known Limitations & Risks
💥

Can Break Without Warning

Google changes internal RPC endpoints at will. A UI update can silently break the library overnight.

🐢

Slow Generation

Audio, video, and slide decks may take 2–30+ minutes. There's no way to speed this up — it's Google's infrastructure generating content.

🚧

Rate Limits

Heavy usage will get throttled. The library has --retry N with exponential backoff, but can't bypass Google's hard limits.

🔐

Cookie Expiry

Google session cookies expire. When they do, you must re-run notebooklm login interactively — no way to automate this without a browser.

📬

No Official Support

This is a community project. If Google changes something, you're waiting on the open-source maintainers to fix it.

🌐

Language is Global

The language setting affects all notebooks in your account simultaneously. No per-notebook language configuration.

What the API Can Do vs. the Web UI

API-Only Features (not in web UI)

✓ Batch downloads — download all artifacts of a type at once with --all
✓ Quiz/Flashcard export — structured JSON, Markdown, or HTML (UI only shows interactive view)
✓ Mind map JSON extraction — hierarchical JSON for use in visualization tools like React Flow
✓ Data table CSV export — download as spreadsheet-ready CSV
✓ Source fulltext access — retrieve the indexed text content of any source
✓ Programmatic sharing — add/remove users, set permissions without clicking through UI
✓ AI-powered research auto-import — web/Drive search with bulk source import
✓ Scripted pipelines — fully automatable, schedule with cron, trigger from CI/CD

Features the API Can't Do

✗ Interactive quiz/flashcard UI — API gets the raw data but not the browser-based quiz experience
✗ Real-time audio streaming — podcasts must finish generating before download
✗ Bypassing rate limits — throttling is enforced server-side by Google
✗ Unattended re-authentication — cookie refresh requires browser interaction if session expires
✗ Reliable long-term automation — Google can break endpoints at any time
✗ Concurrent multi-account control — library manages one Google account at a time per instance

Python Script — Batch Generate for All Cases

import asyncio
from notebooklm import NotebookLMClient

NOTEBOOKS = {
    "kelly-cai": "86438ec8-ec42-4f4e-8331-fda2ed649053",
    # add more case notebooks here
}

async def generate_for_case(client, name, nb_id):
    print(f"Generating artifacts for {name}...")

    # Mind map (sync)
    await client.artifacts.generate_mind_map(nb_id)
    await client.artifacts.download_mind_map(nb_id, f".agent/artifacts/{name}/mindmap.json")

    # Audio (async)
    status = await client.artifacts.generate_audio(nb_id)
    await client.artifacts.wait_for_completion(nb_id, status.task_id)
    await client.artifacts.download_audio(nb_id, f"public/artifacts/{name}/podcast.mp4")

    print(f"Done: {name}")

async def main():
    async with await NotebookLMClient.from_storage() as client:
        for name, nb_id in NOTEBOOKS.items():
            await generate_for_case(client, name, nb_id)

asyncio.run(main())
